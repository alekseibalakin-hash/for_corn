# 2048 с любовью — спецификация

> Подарок-миниапка для супруги в Telegram. Ядро — её любимая 2048, поверх —
> слой «достижения → романтические награды». Это документ-источник правды:
> кодинг-сессии реализуют по нему, CTO-сессия ревьюит против него.

## 1. Видение и принципы

- **Ядро 2048 неприкосновенно.** Это её зона отдыха. Механику не усложняем,
  не ломаем привычный геймплей. Романтика — это *слой сверху*, не вместо.
- **Контент отделён от кода.** Все награды, ачивки и тексты живут в JSON-конфигах
  (`content/rewards.json`, `content/achievements.json`). Добавить новую приятность
  = отредактировать JSON, без правки игрового кода и без кодинг-сессии.
- **Тёплый тон во всём.** Не «Game Over», а «Ничего, ещё разок? ❤️». Приветствие
  по имени (из Telegram). Микрокопия — живая и ласковая, это половина впечатления.
- **Живая, а не свалка.** Купоны имеют срок годности. Кошелёк всегда актуальный.
- **Сочность важна.** Анимации разблокировки/раскрытия наград — эмоция, не галочка.
- **Тёплый минимализм.** Классические цифры, тёплая палитра. Никаких тяжёлых
  ассетов в MVP (см. §9).

## 2. Зафиксированные решения

| Вопрос | Решение |
|---|---|
| Выдача наград | Кошелёк купонов со **сроком годности** |
| Сюрприз | Ачивка даёт **случайный купон из своего тира** (или фикс. `rewardId`) |
| Уведомления мужу | **Нет.** Она показывает экран сама → бэкенд не нужен |
| Архитектура | **Чистый клиент**, без сервера и БД |
| Хранение | Telegram **CloudStorage** (+ localStorage fallback для дев-режима) |
| Раскрытие награды | 🌸 мелочь — **тост**, 💎 крупное — **полноэкранный праздник** |
| Анти-грайнд | Кулдаун у челленджей + дневной потолок купонов (см. §5) |
| Личное слово | Необязательное поле `note` на ачивке/награде → текст на экране раскрытия |
| Визуал | Тёплая 2048 + классические цифры (см. §9 «Дизайн») |
| Стек | React + Vite + TypeScript + Tailwind + Framer Motion |
| Хостинг | Vercel / Cloudflare Pages (статика, бесплатно, HTTPS) |
| Бот | Уже создан; служит лаунчером мини-аппы (см. §12) |

## 3. Игровое ядро

- Классическая 2048: поле **4×4**, свайпы/стрелки, спавн 2 (90%) или 4 (10%).
- **Продолжение после 2048** разрешено (она «набивает очки» — ей нужен endless).
- Game over — когда нет ходов. Кнопка «Новая игра» с подтверждением, тёплый текст.
- Текущая партия сохраняется (resume после закрытия миниапки).
- Управление: свайпы (тач) + стрелки (для теста на десктопе). Хаптика Telegram
  на слияниях и разблокировках.

## 4. Отслеживаемые показатели (stats)

Движок ачивок читает эти поля. **Per-game** сбрасываются с новой партией,
**cumulative** живут вечно.

**Per-game** (объект `currentGame`):
- `sessionScore` — очки текущей партии
- `maxTileThisGame` — макс. плитка в партии
- `movesThisGame` — число ходов
- `gameStartTs` — старт партии (epoch ms)
- `timeToCurrentMaxTileSec` — секунд от старта до появления текущей макс. плитки

**Cumulative** (объект `stats`):
- `totalScore` — сумма очков по всем партиям
- `bestScore` — лучший результат одной партии
- `bestTile` — лучшая плитка за всё время
- `gamesPlayed`, `totalMoves`
- `dailyStreak`, `lastPlayedDate` (YYYY-MM-DD), `firstPlayedDate`
- `rewardsRedeemed` — сколько купонов использовано (для мета-счётчика, см. §6)

Покрывает её цели: новое число (`bestTile`/`maxTileThisGame`), скорость до числа
(`timeToCurrentMaxTileSec`), очки за раз (`sessionScore`/`bestScore`), очки за всё
(`totalScore`).

## 5. Движок ачивок (декларативный, из конфига)

Типы:
- **milestone** — одноразовая веха (256/512/1024/2048/...). Не перевыпускается.
- **challenge** — повторяемый челлендж с `cooldownDays`.
- **surprise** — редкий рандом-дроп. *v2.*

Каждая ачивка задаёт **trigger** — декларативное условие над stats. Операторы
`>= <= > < ==` и составные `allOf` / `anyOf`. Награда: `rewardTier` (случайный
купон из тира — основной путь) либо `rewardId` (конкретный купон). Необязательное
`note` — личное сообщение, которое показывается на экране раскрытия (приоритетнее
`note` у самой награды).

Вычисление: после каждого хода и на game-over прогоняем все активные ачивки по
текущему snapshot stats. Сработавшие → выдаём купон + раскрытие. milestone метим
как `unlocked`, challenge ставим на cooldown.

**Анти-грайнд (супруга амбициозная — может гриндить ачивки):**
- **milestone не фармятся** — каждая один раз (плитки 128…4096, лестница суммарных
  очков 50k/150k/500k). Дневным потолком их **не** режем — пусть первые дни щедрые.
- **challenge** повторяются, но каждый под своим `cooldownDays`.
- **Глобальный дневной потолок** `limits.maxChallengeCouponsPerDay` — максимум новых
  купонов-от-челленджей в сутки, сколько бы она ни играла. Сброс в локальную полночь.
- **Крупные 💎 структурно редки** — только с редких вех и челленджей с длинным
  кулдауном; в ежедневных челленджах их нет.

Схема — см. `content/achievements.json`.

## 6. Система наград и купонов

**Тиры и сроки годности по умолчанию** (переопределяются на уровне награды):

| Тир | Смысл | Срок по умолчанию | Раскрытие |
|---|---|---|---|
| 🌸 small | спонтанная мелочь | 3 дня | тост (не рвёт игру) |
| 💝 medium | требует чуть-чуть планирования | 10 дней | карточка |
| 💎 large | надо планировать заранее | 30 дней | полноэкранный праздник + конфетти |

**Жизненный цикл купона:**
1. Ачивка срабатывает → из пула нужного тира берётся **случайная** награда (или
   фиксированная по `rewardId`) → создаётся купон `{ rewardId, unlockedAt,
   expiresAt }`. Раскрытие — как «открытие подарка/конверта»; если есть `note`,
   показываем твои слова; на 💎 — конфетти (`canvas-confetti`).
2. Купон лежит в кошельке. Карточка показывает таймер до сгорания.
3. За 1 день до конца — мягкая подсветка «скоро сгорит».
4. **На входе в приложение** — бережный баннер, если что-то сгорает в ближайший
   день («купон „Ужин“ сгорает завтра 💝»).
5. **Использовать** → крупный праздничный экран (его она показывает мужу) →
   купон уходит в историю как `redeemed`, `rewardsRedeemed++`.
6. Истёк срок → купон уходит в историю как `expired` (серый, не удаляется молча).

**Мета-счётчик:** где-то ненавязчиво — «ты подарила себе N радостей»
(из `rewardsRedeemed`). Маленькая тёплая гордость, считается из существующих данных.

Проверка сгорания — при открытии приложения и после каждой партии. Время —
epoch ms; доверяем часам устройства.

Схема — см. `content/rewards.json`.

## 7. Данные и персистентность

Хранилище: **Telegram WebApp CloudStorage** (async key-value, на пользователя,
синк между её устройствами). Dev-fallback: localStorage с тем же интерфейсом.

Ограничения CloudStorage: значение — строка ≤ 4096 байт, до 1024 ключей. Храним
несколькими ключами, а не одним мега-объектом:

| Ключ | Содержимое |
|---|---|
| `board` | текущая партия (поле, очки, gameStartTs) для resume |
| `stats` | cumulative-показатели |
| `wallet` | активные купоны |
| `history` | использованные + сгоревшие (обрезаем по длине, напр. 100) |
| `progress` | id milestone, cooldown-таймеры challenge, дневной счётчик (`challengeCouponsToday` + `couponDayDate`) |

Если `wallet`/`history` рискуют превысить 4 КБ — резать историю по длине.

## 8. Telegram-интеграция

- `window.Telegram.WebApp`: `themeParams`, `CloudStorage`, `HapticFeedback`,
  `MainButton`/`BackButton`, `expand()`, `ready()`.
- Идентификация: `initDataUnsafe.user.first_name` — для приветствия по имени.
  Валидация `initData` не нужна (бэкенда нет, чувствительных действий нет).
- Запуск в браузере вне Telegram — работает на mock-SDK (для разработки/ревью).

## 9. Дизайн, тон и источники

Принцип CTO: **дизайн живёт в коде** (`tailwind.config` + токены), без Figma и
без дизайнера. Для клиентской мини-апки-подарка это быстрее, дешевле и полностью
под контролем.

- **Направление:** узнаваемый каркас 2048 + сдвиг палитры в тёплое. Плитки 2048 и
  в оригинале плоские — «без градиентов» нам на руку. Скруглённые углы, мягкие тени.
- **Палитра (токены):** фон-крем `#FBF3EC`, доска `#E7D3C7`, пустая ячейка
  `#F1E4DA`, текст тёплый коричневый `#5B4A42` (приглушённый `#B49A88`),
  акцент-роза/primary `#D4537E`. Веер плиток (персик→коралл→роза→золото):
  2 `#FAEEDA` · 4 `#FAC775` · 8 `#F5C4B3` · 16 `#F0997B` · 32 `#E8895E` ·
  64 `#D85A30` · 128 `#ED93B1` · 256 `#D4537E` · 512 `#993556` · 1024+ золото
  `#854F0B`. Светлый текст на тёмных плитках — `#FBEAF0`.
- **Шрифт:** Nunito (округлый, уютный, отличная кириллица) через `@fontsource/nunito`,
  **self-host** (не дёргаем Google в рантайме). Лицензия OFL.
- **Иконки:** Lucide (`lucide-react`, MIT) — сердце, подарок, часы, кошелёк, кубок.
- **«Иллюстрации» = эмодзи** (🌸💝💎🐕). Бесплатно, нулевой вес, нативный рендер в
  Telegram. Арт-стратегия MVP — без заказа графики.
- **Конфетти:** `canvas-confetti` (MIT) на 💎-наградах.
- **Telegram-тема:** НЕ наследуем полностью (может быть тёмной/чужой). Берём только
  безопасные зоны (`expand()`, viewport, safe-area), палитра — наша фиксированная
  тёплая, чтобы подарок везде выглядел одинаково «нашим».
- **На будущее (v2):** настоящие иллюстрации — unDraw / Storyset (бесплатно,
  перекрашиваются); самый личный ассет — ваши фото собак.

## 10. Стек и структура

```
/                 — конфиг проекта (vite, ts, tailwind)
/src
  /game           — чистая логика 2048 (без UI), покрыта юнит-тестами
  /engine         — движок ачивок + жизненный цикл купонов (юнит-тесты!)
  /storage        — обёртка CloudStorage с localStorage-fallback
  /telegram       — обёртка WebApp SDK + mock
  /ui             — экраны и компоненты (доска, кошелёк, раскрытие, история)
  /content        — загрузка/типизация JSON-конфигов
/content
  rewards.json        — каталог наград
  achievements.json   — правила выдачи
```

Стек: React + Vite + TypeScript, Tailwind, Framer Motion, lucide-react,
canvas-confetti, @fontsource/nunito. **Логика игры и движок наград — чистые функции
с юнит-тестами** (ревью в первую очередь; UI проверяем глазами/в Telegram).

## 11. MVP — что строит первая кодинг-сессия

Линия отреза MVP (всё ниже — обязательно):

- [ ] Скаффолд (Vite + React + TS + Tailwind + Framer Motion + Nunito), запуск локально.
- [ ] Рабочая 2048: 4×4, свайпы+стрелки, очки, game over, продолжение после 2048.
- [ ] Чистая логика игры в `/game` + юнит-тесты (слияния, спавн, конец игры).
- [ ] Stats: per-game + cumulative (вкл. `rewardsRedeemed`), сохранение в storage.
- [ ] Движок ачивок по `achievements.json` (milestone + challenge, без surprise) + тесты.
- [ ] Награды: случайный купон из тира / `rewardId`, кошелёк, срок годности,
      использование, история (redeemed/expired).
- [ ] **Раскрытие:** 🌸 тостом, 💎 полноэкранным праздником + `canvas-confetti`;
      показ `note`, если задан; стиль «открытие подарка».
- [ ] **Тёплый тон:** приветствие по имени (Telegram), ласковая микрокопия, тёплый
      game-over.
- [ ] **Напоминание на входе** о купонах, сгорающих в ближайший день.
- [ ] **Мета-счётчик** «подарено N радостей».
- [ ] Telegram WebApp обёртка + mock для браузера. CloudStorage с fallback.
- [ ] Тёплая палитра по §9, базовая хаптика.

## 12. Деплой и запуск

- Статику деплоим на **Vercel** (или Cloudflare Pages) — бесплатно, HTTPS из коробки.
- **Бот уже создан.** Привязываем к нему задеплоенный URL через @BotFather:
  - либо *Bot Settings → Menu Button → URL* (кнопка-меню в чате бота);
  - либо `/newapp` → Direct Link Mini App (`t.me/<bot>/<app>`) — удобно для прямой
    ссылки/иконки.
- **Токен бота в клиент не попадает** — бэкенда нет, секретов нет.
- Порядок: собрать → задеплоить → получить HTTPS URL → привязать в BotFather →
  открыть у неё.

## 15. Итерация 2 — правки после CTO-ревью

Уточняет/заменяет модель §5-6. Бриф для кодинг-сессии — `briefs/mvp-iter2.md`.

- **Обращение:** ласковое имя «Кукурузка» (константа `PET_NAME`), приветствие всегда
  «Привет, Кукурузка ❤️» (имя из Telegram не показываем — это персональный подарок).
- **Награду надо «Забрать»:** убираем авто-тосты; каждая награда — карточка раскрытия
  с кнопкой «Забрать», несколько за раз — очередью (ни одна не теряется). Тир влияет
  на сочность (🌸 карточка · 💝 побогаче · 💎 полный экран + конфетти). «Забрать»
  (прочитала → купон в кошелёк) ≠ «Использовать» (потратить → задание засчитано).
- **Никакой награды до первого хода:** убрать `evaluateAchievements` из `init`;
  welcome (gamesPlayed≥1) срабатывает после первого хода.
- **Сложность:** пороги в `achievements.json` подняты (партия 6k/15k/30k, сумма 100k/300k/1M).
- **Онбординг:** один раз при первом запуске, короткий текст с интригой (без спойлеров),
  флаг персистится; порядок: онбординг → поле → первый ход → welcome.
- **Память:** резюме обязано работать в реальном Telegram (CloudStorage ≥6.9) — продолжает,
  а не с нуля; проверить путь, не только mock.
- **Единый жизненный цикл заданий (главное):** задание «пройдено» НАВСЕГДА только при
  ИСПОЛЬЗОВАНИИ купона (→ не выпадает больше); при СГОРАНИИ купона задание снова доступно
  (challenge — после `cooldownDays`, milestone — на следующей подходящей партии); пока
  живой купон в кошельке — не дублируется. В состоянии: `unlockedMilestones` → `completed`
  (id заданий с использованным купоном); «pending» определяем по кошельку. Касается и
  milestone, и challenge. Кулдаун и дневной потолок остаются темп-ограничителями.
- **Победа:** когда все задания из конфига в `completed` — баннер «Ты прошла всё! Если
  понравилось — попроси мужа выпустить обновление». Один раз.
- **Мелочи:** ненавязчивый прогресс «выполнено X из N заданий»; в случайном выборе из тира
  не выдавать награду, которая уже лежит в кошельке (без дублей).
