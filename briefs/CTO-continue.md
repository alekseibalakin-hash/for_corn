## РОЛИ
- Проект: Telegram Mini App «2048 с любовью» — подарок жене пользователя («Кукурузка»). Хаб казуальных
  игр (**2048 + Match-3 + «5 букв» Wordle**), за достижения падают купоны на реальные «приятности» от мужа.
  Игра для ОТДЫХА. Чистый клиент (React+Vite+TS+Tailwind+Framer Motion), без бэкенда, GitHub Pages.
- **Муж = продакт-овнер:** решает продукт, тестирует, даёт go/no-go, бампит `?v` в @BotFather.
- **Ты (CTO):** пишешь брифы (briefs/*.md), кодинг отдаёшь в ОТДЕЛЬНЫЕ сессии по брифу, НЕЗАВИСИМО ревьюишь
  работу кодера (сам гоняешь typecheck/test/build; для высоких ставок — адверсариальные мульти-агентные
  workflow), делаешь git commit+push в main, гейчишь деплой, ведёшь память.

## КРИТИЧНО (живые данные жены в проде)
- `STORAGE_VERSION='3'` НИКОГДА не поднимать (сотрёт её прогресс). Миграции строго аддитивные.
- Storage-ключи ТОЛЬКО `[A-Za-z0-9_-]` — **точка невалидна для Telegram CloudStorage** (был прод-баг: ключи
  с точкой молча не сохранялись в Телеге; в браузере работало → невидимо). См. `love2048-ops`. Координатор
  записи `backends.ts` (coalescingStore) не убирать.
- **Деплой:** ты `git commit+push` в `main` → GitHub Actions собирает на Pages-URL (жена на старом `?v` не
  задета) → муж смотрит боевой URL глазами → муж бампит `?v` в @BotFather. Откат — `git revert+push`.

## ⏳ ЧТО В РАБОТЕ ПРЯМО СЕЙЧАС (твой первый шаг)
Кодер (отдельная сессия) делает **§A-v3 + §B по `briefs/match3-polish-features.md`** (→ BUILD_TAG v15).
По его отчёту твой первый шаг — **независимый СТРОГИЙ ревью + адверс-пасс ПЕРЕД пушем** (это её кошелёк И
данные глубины). Содержимое батча:
- **#0 (КРИТИЧНО — баг «48→22»):** durability глубины. Жена дошла до уровня 48, перезашла → ~22.
  `maxSpicyLevel` пишется в CloudStorage асинхронно (троттл+флаш-на-закрытии) → Telegram теряет
  ещё-не-сброшенное. Фикс: localStorage-зеркало глубины ПРАВИЛЬНО — **привязка к Telegram user-id**, потолок
  ≤500, ТОЛЬКО число (не снимок доски), СИНХРОННАЯ запись на победе, read-repair `max(cloud, зеркало)` на
  загрузке, чистка в `resetState`/`?reset=1`. (Это редизайн зеркала «A2», которое раньше убрали из-за 3
  HIGH-багов — теперь без них.)
- **#0.5:** лимит на ВСЕ тир-награды (small/medium/**large**), экземпт ТОЛЬКО **именные** (`rewardId`:
  restaurant/fine-dining). Предикат `isEasyCoupon` → `!achievement.rewardId`. Решает «много призов» (14
  large шли мимо лимита). Крупные не жгутся (перевыдадутся назавтра).
- #1 `grant` в try/catch (чтобы пустой тир не ронял ход); #2 мягкий вход L1 (casual ~85%, через РАСКЛАДКУ
  не бюджет); #3 честный casual-винрейт-тест (де-циркуляризовать, ≥50 трайлов, проверять коридор); #4
  негативные тесты гардов (пустой тир / именная в пуле).
- **§B:** «потратить желание» на ретрай (тратит ОДИН small/medium купон → +5 ходов, продолжить раунд; large/
  именные не тратятся) + реверс-подарок (≤1/сутки: «Вижу, как сильно тебе нравится моя игра 🥰 Значит,
  теперь с тебя презентик 😏»).

**Ревью-чеклист §A-v3+§B:** миграция `progress` аддитивна (кошелёк/`completed`/`rewardsRedeemed` целы);
durability-зеркало = user-id + потолок + reset-чистка + ТОЛЬКО число (адверс-пасс A2 нашёл именно
ретро-выдачу купонов / чужую-доску / reset-дыру — не повтори); новый лимит реально капает large, именные
сверх; `spendCouponForRetry` не растит `rewardsRedeemed` и не трогает large/именные; STORAGE_VERSION цел.

## ПРОЦЕСС-УРОКИ (соблюдать строго)
- **Кодер склонен сам коммитить+пушить.** Для изменений ЕЁ данных/кошелька — ГЕЙТ: ревью ПЕРЕД пушем. В
  брифе прямо «⛔ НЕ коммить/не пушь — сначала отчёт и ревью».
- **Адверс-пасс окупается.** Мульти-агентный workflow (линзы attack→verify) ПЕРЕД пушем поймал то, что
  ручной ревью и кодер пропускали: 3 HIGH-бага на кошелёк, дыры в durability, дубль наград, anti-kindness
  бюджетов. Паттерн: писать `.claude/wf-*.mjs` (gitignored), `node --input-type=module --check` перед
  запуском (частая ошибка — несбалансированные скобки в `parallel(...map(...))` — закрывать `))`), запуск
  в фон, дёрнут по завершении; результат вытаскивать `jq` из output-файла (часто обрезается в нотификации).
- **Тесты гоняют memoryBackend** (не реджектит, не Telegram) → класс «потеря на транзиентном сбое чтения /
  Telegram CloudStorage-специфика (точки в ключах, async-флаш, 4КБ-лимит)» тестами НЕ ловится — только ревью.
- Холодному кодеру — промпт с ориентиром (проект, состояние дерева, не трогать git) + «прочитай HANDOFF +
  бриф». Та же папка, ОДНА сессия на дереве за раз (всё незакоммичено, веток нет → параллельные подерутся).
