## §A. Срочные прод-баги (делать первыми, deploy с v13 как v14)

### A1. 🔑 Ключи с точкой невалидны для Telegram CloudStorage
**Причина (определённая):** Telegram CloudStorage разрешает в ключах только `A-Za-z0-9_-`. Ключи
`match3.board / match3.stats / w5.daily / w5.stats` содержат `.` → запись в Телеге **молча падает**
(error-callback глотается в `.catch`). В браузере (localStorage) точки ок → баг невидим. Поэтому
match3 (лайт+спайси) и Wordle прогресс в Телеге не сохраняются; 2048/кошелёк (ключи без точек) — целы.

**Фикс:** в `src/storage/types.ts` переименовать значения: `match3.board→match3_board`,
`match3.stats→match3_stats`, `w5.daily→w5_daily`, `w5.stats→w5_stats`. Обновить их в `resetState`
(`repository.ts`), убрать стейл-комменты «зарезервировано». **STORAGE_VERSION НЕ трогать.** Миграция
не нужна (под точечными ключами в Телеге данных нет; ценные ключи без точек не трогаем).
**Тест:** все значения `STORAGE_KEYS` матчат `/^[A-Za-z0-9_-]+$/`.

### A2. Подстраховка редкой записи глубины
Даже с верными ключами `maxSpicyLevel` пишется раз-в-уровень (на победе) и висит на ненадёжном
async-flush при закрытии аппы → может не долететь. **Фикс:** на победе (finishSpicyMove, `won`) и в
spicy-ветке `persistBoard` дублировать `maxSpicyLevel` и `.spicy`-снимок **синхронно** в
`window.localStorage` (best-effort, try/catch). На загрузке — read-repair: `maxSpicyLevel =
Math.max(CloudStorage, localStorage-зеркало)`, `.spicy` — свежее из двух. Плюс вызывать `persistStats()`
при выходе из спайси (`onExitToModes`/`onBack`), не только на победе.

### A3. Бюджеты ходов абсурдно раздуты (→ нет проигрыша + слишком легко)
L1 даёт в среднем ~127 ходов на 3–5 льдин → жена всегда побеждает до конца ходов, попап поражения не
появляется, сложности нет. Лечить в `levels.ts buildLevel`:
1. **Абсолютный потолок, привязанный к цели:** `movesBudget = min( max(worst+floor, ceil(worst×mult)), target×K )`, `K≈8` (ручка в `spicy.json` по бэндам).
2. **Убрать раздувание witness:** fallback в `pickSolverMove`/`pickLastIceSolverMove` должен предпочитать ходы, продвигающие лёд (а не первый валидный своп — солвер «гуляет» и раздувает worst).
3. Снизить `budgetMultiplier` ранних бэндов (3.5 → ~1.8–2.0). **`SOLVE_CAP=300` НЕ трогать.**
Цель: L1 ~20–40 ходов, реальный челлендж, проигрыш достижим, победа комфортна. Перегнать `levels.test`
(проходимость сохраняется) + тест «budget − witness разумен (не сотни) для альт-стратегии».

### A4. Дубль именных наград («ресторан» дважды)
1. **Контент:** в `content/achievements.json` у `m3-spicy-25` сменить `rewardId` с `restaurant` на
   `fine-dining` (уже есть в `rewards.json`, tier large — «Ужин высокой кухни», для «вершины перчинки»).
2. **Хардненинг** (`src/engine/achievements.ts`): ПОСЛЕ pending-проверки (стр. ~151) добавить:
   `if (achievement.rewardId && liveRewardIds.has(achievement.rewardId)) { skipped.push({ id: achievement.id, reason: 'pending' }); continue; }`
   — «есть live с этой наградой → такую же не выдавать» (перевыдастся после погашения/сгорания).
3. **Валидатор** (`src/content/index.ts`): предупреждать (НЕ throw — подарок не должен падать), если две
   ачивки имеют одинаковый `rewardId`.
**Тест:** две ачивки на один `rewardId` → выдаётся ОДИН купон, пока он live.
(Два «ресторана» в кошельке жены — оставляем как есть, хирургию не делаем.)

**DoD §A:** typecheck/test/build зелёные; 296+ тестов; STORAGE_VERSION цел; её wallet/completed/2048
целы; BUILD_TAG в `src/ui/constants.ts` → `v14`. Отчёт: правки по файлам, числа бюджетов до/после на

## §B. Фичи (ВТОРЫМ коммитом, только после ревью §A)

### B1. «Потратить желание» на ретрай уровня
В оверлее поражения (`LevelLose`, `Match3.tsx`) — кнопка **«Потратить желание → +5 ходов, продолжить
раунд»** рядом с «Заново». Тратит ОДИН купон тира **small или medium** из кошелька (large/именные НЕ
тратятся — они важные). Механика:
- Новый метод в `RewardsProvider`: `spendCouponForRetry(): boolean` — берёт первый live small/medium
  купон, убирает из `wallet` → в `history` с пометкой «потрачено на продолжение» (НЕ `completed`,
  `rewardsRedeemed` НЕ растёт), персистит. Возвращает успех.
- В `useMatch3`: при успехе — `movesLeft += 5`, `status='playing'`, **доска и прогресс уровня
  СОХРАНЯЮТСЯ** (не регенерим). Нет подходящего купона → кнопка скрыта/задизейблена.
- Это сток купонов — возвращает наградам ценность.
**Тест:** берёт small/medium и не трогает large/именные; `rewardsRedeemed` не растёт; купон уходит в историю.

### B2. Реверс-подарок (раз в сутки максимум)
Когда жена за день много играет (выбрала дневной лимит купонов И сыграла ещё ≥N партий) — РАЗ В СУТКИ
показать тёплую модалку:
> **«Вижу, как сильно тебе нравится моя игра 🥰 Значит, теперь с тебя презентик 😏»**

Просто сообщение (без купон-механики). Трекать аддитивно в `progress`: дневной счётчик партий + дата
«показано сегодня». STORAGE_VERSION не трогать.

**DoD §B:** тесты на `spendCouponForRetry` и на «реверс ≤1/сутки»; гейт зелёный; BUILD_TAG → `v15`
(или `v14`, если §A+§B одним деплоем — решит CTO).
