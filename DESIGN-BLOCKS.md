# «Блоки-фигуры» — дизайн 4-й игры хаба (поуровневый Block Blast)

> Аддендум к DESIGN.md / DESIGN-HUB.md (не заменяет). 4-я игра хаба «2048 с любовью» рядом с
> 2048 + Match-3 (лайт/спайси) + «5 букв». Источник правды для кодинг-сессии; CTO ревьюит против него.

## 0. Видение и ключевое решение
Казуальная пространственная головоломка «составь фигуру» (Block Blast / Woodoku-стиль), но **НЕ
эндлесс-релакс, а ПОУРОВНЕВАЯ прогрессия «сложнее и сложнее»** — по образцу match3 «с перчинкой».
Причина — профиль игрока (см. memory `love2048-player-taste`): у жены лидер-дух, она хочет проходить
нарастающие уровни (дошла до L48 в спайси); бесконечные простые раунды ей скучны.

**Решения мужа (2026-06-19):** цель уровня = **«расчистить поле»**; режим = **только поуровневый**
(без релакса в MVP).

## 1. Механика ядра
- Поле **8×8** (как match3 — переиспользуем сетку/палитру/тач-метрики).
- Снизу — **набор из 3 фигур** (полимино 1–5 клеток: уголки, линии, квадраты, L/T/S-формы).
- **Drag-and-drop** фигуры на поле (Framer Motion `drag`); валидная позиция подсвечивается, невалидная —
  не ставится. Фигуру нельзя вращать (классика Block Blast — сложность в подборе места, не в верчении).
- Разместил все 3 → выдаётся новый набор (детерминированно по seed).
- Когда **полный ряд ИЛИ столбец** заполнен — он **сгорает** (освобождается). Несколько линий за одно
  размещение → комбо-множитель очков + «сочный» праздник (переиспользуем конфетти/flash из match3).
- **3×3-квадраты (Woodoku) — НЕ в MVP** (усложняют; ряды/столбцы чище для старта).

## 2. Цель уровня — «расчистить поле»
- На старте часть клеток занята **особыми блоками** (тёплый стиль — напр. «карамельки»/«камешки»; визуал
  согласуем, переиспользуя слой обстаклов match3 как референс).
- Особый блок: (а) считается заполненной клеткой для сжигания линии; (б) **исчезает**, когда сгорает его
  ряд/столбец (= прогресс цели); (в) на него нельзя ставить фигуру.
- **Цель уровня = расчистить ВСЕ особые блоки** за **лимит наборов фигур** (аналог лимита ходов спайси).
- Чтобы убрать блок — заполнить остальную его линию обычными фигурами и сжечь. Это и есть головоломка
  (близко к её любимому `clearIce`).

## 3. Прогрессия и фейл-стейт (модель спайси, проверена)
- **Лимит «наборов фигур»** на уровень — мягкий фейл-стейт. Проигрыш = лимит исчерпан и не расчищено
  (или ни одна из 3 текущих фигур никуда не влезает — тупик).
- **Добрый бесконечный ретрай** (тот же уровень, новый seed, та же сложность) — «Почти! Ещё разок ❤️».
- **Глубина (`bb_maxLevel`) durable** и НЕ теряется при проигрыше/выходе.
- Нарастающая сложность глубже: больше особых блоков, теснее старт, неудобнее раздача фигур, туже лимит.
  Балансить **с запасом вверх** — она мастеровитее «casual»-модели.
- Опционально позже: «потратить желание» (+1 набор/переброс фигур) — как §B1 в match3. НЕ в MVP.

## 4. 🔑 Генератор честных уровней + солвер-гейт (ГЛАВНАЯ инженерная задача)
По образцу спайси (`levels.ts`) — но это самостоятельный мини-проект, главный риск проекта.
- **Seeded** всё: `mulberry32` (из `src/engine/rng.ts`) — стартовый расклад блоков + **поток выдаваемых
  фигур** + позиция потока персистятся. Без seed гарантия проходимости недействительна (урок спайси №0).
- **ПРАВИЛО №1: уровень обязан быть проходим** — конструктивная генерация → **жадный/поисковый
  `simulateSolve`** (DFS с откатом по размещениям; эвристики: сначала ходы, дающие клир/убирающие блок) как
  доказательство в пределах лимита наборов → reject, если солвер не решил → **parachute-fallback**
  (total-функция, всегда возвращает проходимый уровень). Тест: уровни 1..N × seeds проходимы.
- **Потолок честности:** глубже = выше цель/туже, НЕ «нечестно тесно».
- ⚠️ Перф: солвер размещений полимино дороже match3-солвера (ветвление по фигура×позиция). Нужен бюджет
  глубины поиска + мемоизация; целевое время генерации уровня — десятки мс (живой переход без лага).
- Этот раздел заслуживает отдельной проработки (как спайси — несколько подходов + судья).

## 5. Хранилище и durability (бережно — урок love2048-ops)
- Новые ключи **`bb_board` / `bb_stats`** — **БЕЗ точек** (`/^[A-Za-z0-9_-]+$/`; точка молча ломает
  Telegram CloudStorage — выученный прод-баг v14). Аддитивно, **STORAGE_VERSION='3' НЕ трогать**.
- `bb_stats`: cumulative (`bb_maxLevel` монотонный, score/games/combos). `bb_board`: незаконченный уровень
  (level/seed/streamPos/goalProgress/setsLeft/доска/блоки) для резюма.
- **Durability глубины:** переиспользовать паттерн `depthMirror` (синхронное localStorage-зеркало числа,
  привязка к Telegram user-id, потолок, `max(cloud,зеркало)` на загрузке, чистка в reset). Обобщить
  `depthMirror` на неймспейс (`spicy_depth_*` / `bb_depth_*`) ИЛИ продублировать — РЕШИТЬ в брифе.

## 6. Награды (под общий слой, экономику не раздуть)
- Игро-независимый наградный слой как есть (`rewards.grant('bb', снапшот, prev)`). Неймспейс статов
  **`bb_*`**, ачивки `game:'bb'`.
- Вехи по глубине `bb_maxLevel` (нелинейные пороги — нет грайнда) + несколько per-game (счёт/комбо-линии).
- **ВАЖНО:** новые «капируемые» ачивки (без `rewardId`) попадают под уже введённый дневной лимит
  (`isCappedCoupon`) — частоту купонов не раздуваем (это была цель v14). Именная веха-вершина — максимум 1.
- `content/rewards.json` пул не трогаем; добавляем только `achievements.json` (+ при желании 1 именную).

## 7. Архитектура / плагин в хаб
- Новая плитка в `Hub.tsx`; роутинг `view: 'bb'` (как 'm3'); lazy-load (`React.lazy`) — бандл не растёт.
- Чистая логика `src/games/blocks/`: `logic.ts` (fits/place/clearLines/isDead/score — БЕЗ id, юнит-тесты),
  `gems.ts`-аналог при нужде для анимации, `levels.ts` (генератор+солвер+parachute), `useBlocks.ts`
  (зеркало `useMatch3`: поуровневый персист dual-slot не нужен — один режим), `Blocks.tsx` (UI+drag+HUD+оверлеи).
- Тон/арт: тёплые блоки/эмодзи, ласковая микрокопия, добрый фейл. Хаптика на клире/празднике.

## 8. Фазы и DoD
- **Фаза 1 — ядро + генератор:** логика (place/clear/dead) + генератор честных уровней + солвер-гейт + тесты
  (проходимость 1..N×seeds, parachute-тотальность, детерминизм). Это костяк и главный риск.
- **Фаза 2 — UI + плагин:** drag-drop, HUD (уровень/цель/наборы), оверлеи победы/поражения, плитка хаба,
  награды `bb_*`, durability-зеркало, персист `bb_board/bb_stats`.
- **DoD:** typecheck/test/build зелёные; каждый уровень доказуемо проходим (тест); durable глубина (тест на
  read-repair); STORAGE_VERSION цел; ключи без точек; лайт/2048/match3/wordle не задеты; экономика под лимитом.
