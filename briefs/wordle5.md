## §0. Суть
Wordle на русском. Два режима: **ежедневный** (слово дня по локальной дате) и **бесконечный**. Третья
плитка хаба рядом с 2048 и Match-3. Награды/кошелёк — позже (СТО), сейчас игра изолирована.

## §1. Механика
- Поле 5×6 (5 букв × 6 попыток). Каждая попытка — **полное новое 5-буквенное слово**; буквы не залипают,
  игрок печатает все 5 каждый раз (найденные подсвечены в прошлых строках и на клавиатуре — это память,
  а не автоввод).
- Раскраска (после сабмита): зелёный=на месте, жёлтый=есть но не там, серый=нет. Клавиатура — лучший цвет.
- Победа: 5 зелёных за ≤6. Поражение: 6 попыток → показать слово. Повтор слова не блокируем.
- **Ё→Е**: клавиатуры с Ё нет; всё нормализуется ё→е (как у Яндекс/Тинькофф «5 букв»).

## §2. Словарь
| Список | Размер | Назначение |
|---|---|---|
| `answers.json` | ~3–5к | Загаданные. Только нейтральные нарицательные сущ. им.п., без имён/мата/мусора |
| `allowed.json` | ~10–30к | Любой допустимый ВВОД (шире; падежные формы ок). Нет в списке → встряхнуть, не принять |
Файлы в `content/wordle/`. Детали подготовки и чистки — §12.

## §3. Режимы
**Ежедневный:** `dateKey` = номер **локального** дня (`Math.floor((Date.now() − getTimezoneOffset()×60000)/86_400_000)`); индекс = `dateKey % answers.length`. Одна партия в день, сброс в локальную полночь, переиграть нельзя (показываем результат).
**Бесконечный:** случайное слово (`Math.random`) каждый старт, переиграть сразу можно. Статы отдельно.

## §4. Состояние и персист
```ts
interface W5DailyState { dateKey: number; guesses: string[]; status: 'playing'|'won'|'lost'; }
interface W5Stats { dailyPlayed: number; dailyWins: number; endlessPlayed: number; endlessWins: number; bestGuess: number; }
```
Ключи `w5.daily` / `w5.stats` — **аддитивно** в `STORAGE_KEYS` + методы в `repository.ts` (через общий
`loadJSON`) + в `resetState`. `STORAGE_VERSION` НЕ трогать. Нормализаторы `normalizeW5Daily`/`normalizeW5Stats`
(мягкое чтение; старое отсутствие → дефолт). daily перезаписывается при смене `dateKey`.

- Сетка 5×6, переворот тайлов при раскрытии (CSS flip по очереди), встряхивание строки на невалидном вводе,

- Финал-оверлей: результат + слово + «Ещё»(endless)/«Завтра»(daily).
- **Экрана статистики/распределения попыток НЕТ** (владелец отказался). Статы только копятся в стораже.

## §7. Алгоритм раскраски (edge-case дублей)
Два прохода: (1) пометить все зелёные (точное совпадение), убрать их из пула букв ответа; (2) по не-зелёным:
буква есть в пуле → жёлтый (убрать из пула), иначе серый. Корректно для дублей (ААААА против КАША → одна А
жёлтая, остальные серые).

## §9. Тесты
Раскраска (все дубли) · Ё→Е · daily-детерминизм · персист round-trip + normalize · невалидный ввод не принят
· **гард данных**: бэкенд с reject getItem → загрузка НЕ перезаписала сохранёнку дефолтом.

## §10. Зафиксированные решения (бывшие открытые вопросы)
1. Режимы — **оба** (daily + endless).
2. Хард-мод — **нет** (для подарка излишне).
3. Экран статистики Wordle-стиля — **нет** (статы копим внутренне для будущих ачивок).
4. Клавиатура — **своя** в игре (нативную нельзя: прячет поле, не красит клавиши; так у всех Wordle).
5. Анимации — **переворот тайлов + встряхивание + прыжок** «как у Тинькофф».
6. Словарь — **готовая открытая база, почищенная** (§12), с ручным ревью списка ответов перед деплоем.
7. Награды/кошелёк — **нет** сейчас (СТО прикрутит позже; `w5_` статы копим).
