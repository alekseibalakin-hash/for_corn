# HANDOFF — «2048 с любовью» (Telegram Mini App, хаб игр-подарок)

Цель этого файла — чтобы новая сессия подхватила проект бесшовно. Прочитай также:
`DESIGN.md` (база), `DESIGN-HUB.md` (хаб/match3), `MEMORY.md` + memory-файлы (автозагрузка),
`briefs/*.md` (история ТЗ), `content/*.json` (контент).

## Что это
Подарок жене (ласково — «Кукурузка») в Telegram. Хаб казуальных игр (сейчас 2048 + Match-3),
где за достижения выпадают КУПОНЫ на реальные «приятности» от мужа (ужин в ресторане, прогулка
с собаками вместо неё, день без домашних дел…). Игра для ОТДЫХА — без лимитов ходов/проигрыша/
стресса. Чистый клиент, без бэкенда.

## Роли (сохрани эту модель — пользователь хочет «как было»)
- **Пользователь (муж)** — продакт-овнер/заказчик. Решает продукт, пишет/одобряет тексты наград,
  делает шаги на СВОИХ аккаунтах (GitHub-репо; @BotFather — смена `?v=` для кэш-бастинга).
  Тестирует и даёт go/no-go перед выкаткой к жене. Любит скорость, но бережёт ЖИВЫЕ данные жены.
- **Ты (ассистент) = CTO + ассистент.** Докручиваешь идею, проектируешь, пишешь спеки/брифы.
  Кодинг обычно отдаётся в ДРУГИЕ кодинг-сессии (муж вставляет твой бриф) — но мелкие/срочные
  правки делаешь сам. НЕЗАВИСИМО ревьюишь работу кодера (сам гоняешь typecheck/test/build; для
  высоких ставок — адверсариальные мульти-агентные workflow). Гейтишь деплой. Делаешь git
  (commit+push в `main`), следишь за Pages, говоришь мужу про `?v`. Ведёшь память.
- **Язык — русский.** Тон — CTO: конкретно, рекомендация а не опросник. С её живыми данными —
  параноидально осторожно (сначала доказать безопасность, потом катить).

## Наградная модель (§15 — ключевое)
- Триггер ачивки → КУПОН в ОБЩИЙ кошелёк. Награда: случайная из тира (🌸 small/💝 medium/💎 large)
  или фикс `rewardId` (только `reach-2048 → restaurant`).
- Жизненный цикл: «пройдено НАВСЕГДА» только при ИСПОЛЬЗОВАНИИ купона (redeemed); сгорел → снова
  доступно; живой купон в кошельке → не дублируется (pending).
- Счётчик «выполнено X из N» = ИСПОЛЬЗОВАННЫЕ (redeemed), НЕ заработанные. By design, не «чинить».
- Анти-грайнд: milestone без лимита; challenge — cooldownDays + дневной потолок
  (`limits.maxChallengeCouponsPerDay=3`, сброс в локальную полночь). Пороги — ручка в JSON.
- **Edge-triggering**: per-game вехи (maxTileThisGame/sessionScore/timeToCurrentMaxTileSec/
  m3_score/m3_combo/m3_moves/m3_biggestClear) выдаются ТОЛЬКО при пересечении порога ходом
  (через `prevSnapshot`) — иначе резюм партии ронял купон на первом ходу. Кумулятивные/глобальные
  (totalScore/totalMoves/gamesPlayed/dailyStreak/rewardsRedeemed/m3_combos…) — level (`>=`).
- Призы: ОБЩИЙ пул (80) и ОБЩИЙ кошелёк на обе игры. Нет одновременных дублей в кошельке
  (глобальный дедуп по live-купонам); со временем призы повторяются (пул не «расходник»).
- Неймспейс статов: 2048 — без префикса; match3 — `m3_*`. Ачивка имеет `game` (`'2048'`/`'m3'`/
  `'any'`; untagged → '2048'); движок фильтрует по активной игре + мержит global-статы.

## ДАННЫЕ И БЕЗОПАСНОСТЬ (критично — НЕ сломать прогресс жены)
- Хранилище: Telegram CloudStorage (≥6.9), fallback localStorage. Ключи: board/stats/wallet/
  history/progress/schemaVersion + match3.board/match3.stats. wallet/history/progress — ОБЩИЕ.
- **`STORAGE_VERSION='3'` при смене СБРАСЫВАЕТ ВСЁ. НИКОГДА не поднимай.** Миграции — только
  аддитивные (см. seedGlobals/normalize*).
- `?reset=1` в URL — ручной полный сброс; боевая ссылка жены его НЕ содержит.
- **Координатор записи** (backends.ts `coalescingStore`): коалесинг по ключу + сериализация
  (last-write-wins) + троттл + flush при сворачивании аппы + read-your-writes. Это ФИКС бага
  потери прогресса (CloudStorage троттлил частые записи 2048 → за сессию терялись награды). Не убирать.
- Анимация match3: id ЖИВУТ ТОЛЬКО в gems.ts; logic.ts/Gem/Board БЕЗ id; персист — plain board без
  id. Инвариант `gems.test`: `gemsToBoard(applyStep(prev,step)) === step.board` — замок от десинка.

## Версии и деплой
- BUILD_TAG (constants.ts) — видимая метка в футере. Кэш-бастинг — `?v=N` на URL кнопки в
  @BotFather (Telegram кэширует по URL).
- Деплой: `git push` → main → Actions → Pages (~1 мин). Проверка: `curl` главной, смена хеша
  `assets/index-*.js` = новая сборка жива.
- ВЫКАТКА К ЖЕНЕ двухступенчатая: (1) push в main → сборка живёт на URL (её бот на старом `?v`,
  не задет); (2) муж смотрит на боевом URL глазами; (3) муж бампит `?v` в BotFather → её Telegram
  подтягивает. Откат при беде — `git revert` + push.
