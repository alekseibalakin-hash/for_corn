# Хаб приятностей + Match-3 — дизайн расширения

> Аддендум к DESIGN.md (не заменяет его). 2048 → хаб из нескольких игр, все кормят
> ОДИН банк приятностей. Фаза A: хаб + переиспользуемый наградный слой. Фаза B: Match-3.
> Zuma (Фаза C) — позже, отдельным решением.

## 1. Принцип: наградный слой игро-независим

Движок наград читает плоский снапшот (`ключ → число`) + конфиг ачивок. Кошелёк, купоны,
`completed`, дневной потолок, сгорание, раскрытие, victory — всё ОБЩЕЕ. Новая игра = новый
источник статов + новые ачивки в JSON. Каталог из 80 наград переиспользуется как есть.

## 2. Целевая архитектура

```
App
 ├─ Hub            — выбор игры, общий кошелёк, «выполнено X из N» по всему хабу
 ├─ rewards/       — useRewards/RewardsProvider: wallet, history, progress,
 │                   rewardsRedeemed, dailyStreak, evaluate+выдача, redeem→completed,
 │                   очередь раскрытий, victory, expiry, персист ОБЩИХ ключей.
 │                   ИГРО-НЕЗАВИСИМ.
 ├─ games/g2048    — текущая логика 2048, плагинится в rewards
 └─ games/match3   — Фаза B
```

Каждая игра: свой хук строит свой снапшот и зовёт общий `rewards.grant(gameId, snapshot)`
на ходах/смене партии. Раскрытие/кошелёк/победа — из общего слоя.

## 3. Неймспейс статов

Движок оценивает МЕРЖ: `{ ...globalStats, ...снапшот активной игры }`.

- **global** (владеет rewards): `rewardsRedeemed`, `dailyStreak`.
- **2048**: остаются БЕЗ префикса (`maxTileThisGame`, `sessionScore`, `totalScore`,
  `bestScore`, `bestTile`, `gamesPlayed`, `totalMoves`, `timeToCurrentMaxTileSec`) →
  существующие 25 ачивок НЕ трогаем.
- **match3**: с префиксом `m3_` (`m3_score`, `m3_bestScore`, `m3_combo`, `m3_gemsCleared`,
  `m3_gamesPlayed`, `m3_moves`).

Ачивка получает необязательное поле `game`. Движок берёт ачивки, где
`(a.game ?? '2048') === gameId` ИЛИ `a.game === 'any'`. Untagged → `'2048'` (нулевой churn
к существующему конфигу).

## 4. Хранилище и МИГРАЦИЯ (бережно — живые данные жены!)

Текущие ключи: `board, stats, wallet, history, progress, schemaVersion='3'`.

- **ОБЩИЕ — НЕ трогаем, сохраняем как есть:** `wallet`, `history`, `progress`. Её купоны и
  `completed` обязаны выжить.
- **`rewardsRedeemed`** сейчас внутри `stats` (2048). Переносим в общий слой (в `progress`):
  при загрузке, если `progress.rewardsRedeemed` не задан — засеять из `stats.rewardsRedeemed`.
  Лосслесс, одноразово.
- **2048** остаётся на ключах `board`/`stats` (без миграции). match3 — новые ключи
  `match3.board`/`match3.stats`.
- **НЕ поднимать `STORAGE_VERSION`** — это обнуляет всё! Миграция строго аддитивная, без сброса.
- **`dailyStreak`** становится хаб-стриком (играла в любую игру сегодня), владелец — rewards.

## 5. Хаб-экран

Плитки игр (2048 — играбельна; «Match-3 — скоро ✨» — задизейблена в фазе A), вход в кошелёк,
«выполнено X из N заданий» по всему хабу, тёплая палитра/Nunito. Роутинг простым состоянием
`view: 'hub' | 'g2048' | 'm3'` (без react-router), «назад» в хаб. Онбординг — один раз на
входе в хаб. Игры — через `React.lazy` + Suspense (ленивая загрузка), чтобы бандл не рос и
будущая Zuma не утяжелила старт.

## 6. Match-3 (Фаза B)

- Поле 8×8, 5–6 типов (эмодзи-фрукты/сердечки — арт-стратегия «эмодзи», без спрайтов).
- Свайп/тап фишки к соседу → своп; если ≥3 в ряд → очистка, гравитация, добивка сверху,
  КАСКАД с комбо-множителем; если матча нет → откат свопа.
- **Расслабленный endless** (без проигрыша, как 2048 — это отдых); если ходов нет — реш-шафл.
- Чистая логика `src/games/match3/` + юнит-тесты: `findMatches`, `applyGravity`, `refill`,
  `resolveCascades`, `isValidSwap`, `score`.
- Ачивки match3 (`game:'m3'`): первый матч; комбо x4 (каскад ≥4); 5000 за партию; очисти
  1000 фишек суммарно; и т.д. → общий кошелёк.

## 7. Фазы

- **A. Хаб + наградный слой** — рефактор + аддитивная миграция; 2048 плагинится; плитка
  match3 «скоро». Бриф: `briefs/hub-phase-a.md`.
- **B. Match-3** — логика+тесты+UI+ачивки, плагинится в хаб. Бриф — после ревью A.
- **C. Zuma** — позже; полная или упрощённый шутер-попер (решим отдельно).

## 8. Definition of done фазы A (главное)

- Тесты/типчек/сборка зелёные (+ тесты: фильтр ачивок по `game`, merge-снапшот, миграция
  `rewardsRedeemed`).
- МИГРАЦИЯ доказана тестом: старое состояние (`wallet`/`history`/`progress`/`stats` со
  `schemaVersion='3'`, `stats.rewardsRedeemed>0`, купон в кошельке, id в `completed`) →
  после загрузки кошелёк/история/`completed` на месте, «подарено N» = старому значению,
  партия 2048 резюмится.
- 2048 играется из хаба идентично; плитка match3 «скоро».
- 2048 грузится лениво (отдельный чанк).
