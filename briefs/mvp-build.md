# Бриф для кодинг-сессии №1 — MVP

> Скопируй блок ниже в новую сессию этого проекта, чтобы запустить сборку MVP.
> CTO-сессия (эта) ревьюит результат против `DESIGN.md`.

---

```
Ты — разработчик в проекте «2048 с любовью»: Telegram Mini App-подарок для супруги.
Работай ТОЛЬКО внутри этой папки, наружу не выходи.

ПЕРЕД КОДОМ прочитай целиком DESIGN.md — это источник правды. Контент уже готов:
content/rewards.json (80 наград) и content/achievements.json (правила выдачи + блок
limits + примеры личных note). Их НЕ редактируй по смыслу — только грузи и типизируй.

Стек (DESIGN §10): Vite + React + TypeScript + Tailwind + Framer Motion,
lucide-react, canvas-confetti, @fontsource/nunito (self-host).

Собери MVP по чек-листу DESIGN §11:
- Скаффолд, запуск локально.
- Игра 2048: поле 4×4, свайпы + стрелки, очки, game over, продолжение после 2048.
- Чистая логика игры в src/game (без UI) + юнит-тесты: слияния, спавн (2/4), конец игры.
- Stats per-game и cumulative, вкл. rewardsRedeemed (DESIGN §4), через src/storage.
- Движок ачивок src/engine по achievements.json: milestone + challenge, операторы
  >= <= > < == и allOf/anyOf, cooldown для challenge, поле note. Юнит-тесты на движок.
- АНТИ-ГРАЙНД (DESIGN §5): milestone выдаются один раз и НЕ лимитируются; купоны от
  challenge ограничены limits.maxChallengeCouponsPerDay в сутки (счётчик в progress,
  сброс в локальную полночь). Покрой это тестами.
- Награды: ачивка → случайный купон из тира (или rewardId), кошелёк, срок годности,
  использование, история redeemed/expired. Сгорание — при открытии и после партии.
- Раскрытие награды: 🌸 small — ненавязчивый ТОСТ; 💎 large — полноэкранный праздник
  + canvas-confetti; стиль «открытие подарка»; показывай note, если задан.
- Тёплый тон: приветствие по имени (Telegram first_name), ласковая микрокопия,
  тёплый game-over («Ничего, ещё разок?»).
- Напоминание на входе о купонах, сгорающих в ближайший день.
- Мета-счётчик «подарено N радостей» (из rewardsRedeemed).
- Хранилище: Telegram CloudStorage с localStorage-fallback (ключи из DESIGN §7).
- Telegram WebApp-обёртка src/telegram + mock, чтобы работало в обычном браузере.
- Тёплая палитра и токены строго по DESIGN §9 (цвета, Nunito, скругления), хаптика.

НЕ делай (это v2, см. DESIGN §13): бэкенд, уведомления, surprise-дропы, soft-currency,
фото/персонализацию, звук, экран статистики. Деплой и привязку к боту НЕ делай —
это отдельный шаг (DESIGN §12), бот уже создан.

Definition of done:
1. `npm run dev` запускает игру в браузере (через Telegram-mock).
2. Юнит-тесты на src/game и src/engine проходят (`npm test`), включая анти-грайнд.
3. Можно сыграть, получить купон (с раскрытием и note), использовать его и увидеть
   в истории; купон сгорает по сроку; счётчик радостей растёт.
4. Гринд челленджей упирается в дневной потолок; вехи выдаются всегда.
5. Состояние переживает перезагрузку страницы (storage работает).

В конце дай краткий отчёт: что сделано, как запускать, как гонять тесты, что осталось.
```

## Что проверит CTO-сессия после сборки
- Чистота `src/game` и `src/engine`: чистые функции, есть тесты, логика 2048 корректна.
- Движок верно читает `achievements.json` (триггеры, cooldown, milestone-однократность, note).
- Анти-грайнд: дневной потолок на купоны-от-challenge, milestone не лимитируются, сброс в полночь.
- Жизненный цикл купона: случайный из тира, срок годности, использование, сгорание.
- Раскрытие: тост для 🌸, полноэкранный праздник + конфетти для 💎, показ note.
- Тёплый тон: имя, микрокопия, game-over; напоминание о сгорании; счётчик радостей.
- Storage: запись/чтение, fallback вне Telegram, лимиты ключей CloudStorage.
- Дизайн по §9: палитра, Nunito, скругления; «сочность» анимаций.
- Прогон вживую: сыграть → получить → использовать → история → сгорание → упор в потолок.
