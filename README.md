# 2048 с любовью 💝

Telegram Mini App-подарок: её любимая 2048 + слой «достижения → романтические награды».
Источник правды — [DESIGN.md](DESIGN.md). Контент (награды и правила выдачи) живёт в
`content/*.json` и правится без участия кода.

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173 — играбельно в браузере через Telegram-mock
```

Вне Telegram приложение использует mock-SDK (CloudStorage → localStorage, хаптика →
`navigator.vibrate`). Имя для приветствия можно задать через `?name=Имя`
(например, `http://localhost:5173/?name=Любимая`).

```bash
npm run build      # прод-сборка (tsc --noEmit + vite build), Nunito self-host
npm run preview    # предпросмотр прод-сборки
```

## Тесты

```bash
npm test           # vitest run — чистая логика игры и движка наград
npm run test:watch
npm run typecheck  # tsc --noEmit
```

Покрыто (`src/game`, `src/engine`, `src/ui/tiles`, `src/storage`, `src/content`):
слияния и спавн 2/4, конец игры, операторы и `allOf/anyOf`, **единый жизненный цикл
заданий** (DESIGN §15): complete-on-redeem (использован → больше не выпадает),
expire-retry (сгорел → снова доступен), pending (живой купон в кошельке → не дублируется),
разнообразие наград; темп-ограничители (дневной потолок челленджей + сброс в полночь,
кулдаун); жизненный цикл купона (срок годности, использование, сгорание), приоритет
`note`, мягкое чтение прогресса, сверка анимационной модели плиток с чистым ядром,
обрезка истории под лимит CloudStorage.

## Структура (DESIGN §10)

```
content/                 — каталог наград + правила выдачи (НЕ редактировать по смыслу)
src/game/                — чистая логика 2048 (поле, ход/слияние, спавн, конец игры)
src/engine/              — движок ачивок, анти-грайнд, жизненный цикл купонов, stats
src/content/             — типизация и загрузка JSON + валидация целостности
src/storage/             — KVStore: CloudStorage ↔ localStorage ↔ memory + репозиторий
src/telegram/            — обёртка WebApp SDK + mock для браузера
src/ui/                  — экраны: доска, кошелёк, раскрытие, баннеры, хук-оркестратор
```

## Что внутри (MVP, DESIGN §11)

- Классическая 2048 4×4: свайпы + стрелки, очки, тёплый game-over, endless после 2048.
- Stats per-game и cumulative (вкл. `rewardsRedeemed`), сохранение между сессиями.
- Декларативный движок ачивок по `achievements.json`: milestone + challenge.
- **Единый жизненный цикл заданий (DESIGN §15):** задание пройдено НАВСЕГДА только при
  использовании купона; сгорел — снова доступно; пока живой купон в кошельке — не
  дублируется. Темп-ограничители: дневной потолок купонов-от-челленджей (сброс в
  локальную полночь) и кулдаун. Ачивки считаются только на ходах — никакой награды до
  первого хода.
- Кошелёк купонов: случайный из тира (без дублей с тем, что уже на руках) или
  фиксированный, срок годности, использование, история `redeemed/expired`, сгорание
  при входе и после партии. Прогресс «выполнено X из N заданий».
- Раскрытие: КАЖДУЮ награду надо «Забрать» карточкой (без авто-дисмисса, очередью);
  сочность по тиру (🌸 карточка · 💝 побогаче · 💎 полный экран + `canvas-confetti`);
  показ личного `note`. Победный баннер, когда все задания пройдены (один раз).
- Тёплый тон: онбординг при первом запуске, ласковое обращение «Кукурузка», напоминание
  о сгорающих купонах на входе, мета-счётчик «подарено N радостей».
- Тёплая палитра и токены строго по DESIGN §9, Nunito (self-host), базовая хаптика.

Не входит (v2, DESIGN §13): бэкенд, уведомления, surprise-дропы, soft-currency,
фото/персонализация, звук, экран статистики.

## Деплой (отдельный шаг, DESIGN §12)

Собрать → задеплоить статику (Vercel / Cloudflare Pages, HTTPS) → привязать URL к уже
созданному боту через @BotFather (Menu Button или `/newapp`). Токен в клиент не кладём.
